# Custom Site Guide

Create styling and layout in this repo without adding platform responsibilities.

Use `NEXT_PUBLIC_DESIGN_MODE=true` for visual iteration. Use only safe sample data from `src/data/sample-event.json`.

The reusable starter baseline should remain the platform-compatible public wedding renderer. Do not replace the starter default with a marketing shell, alternate navbar, custom hero, client theme, or direct RSVP form.

For a per-client custom repo, fork or copy this starter, confirm the baseline visually matches the main platform preview, then replace styling and assets with client-approved public content. Connect it to a published platform event slug in live mode only after the baseline passes.
